## 你是不是也這樣？

滑手機看到一篇超棒的文章，心想「這我之後一定會用到」，隨手丟進書籤資料夾。結果三個月後真的需要時，翻了半小時也找不到——甚至忘了自己存過。

更慘的是，筆記 App 裡堆了幾百條筆記，標籤亂到連自己都不想打開。每次想整理，看著那堆雜亂的內容就先放棄了。

如果你有過這種經驗，Tiago Forte 的 **PARA 方法**就是為你設計的。

## 什麼是第二大腦？

「第二大腦」（Building a Second Brain）是生產力專家 Tiago Forte 提出的概念。核心想法很簡單：**不要把所有東西都塞進腦袋，把記憶和整理的工作交給外部系統**，你的大腦用來思考就好。

大腦是用來產生想法的，不是用來儲存想法的。

但要讓外部系統真正發揮作用，你需要一個清晰的組織方式。這就是 PARA 登場的時候。

## PARA 四層架構一次搞懂

PARA 是四個英文單字的縮寫，代表四種資訊層級：

### 1. Projects（專案）—— 有終點的任務

專案是**有明確目標和截止日期**的工作。像是：

- 準備六月的 AWS 認證考試
- 規劃八月的家庭旅行
- 寫一份 Q3 行銷企劃

關鍵判斷標準：**能說出「完成」的那一天**，就是專案。

### 2. Areas（領域）—— 需要持續維護的責任

領域是**沒有終點、但需要持續關注**的範圍。像是：

- 健康（永遠不會「完成」健康）
- 財務（持續追蹤收支和投資）
- 人際關係（需要不斷經營）
- 職涯發展（長期方向）

專案和領域的關係：**每個專案通常對應一個領域**。例如「準備 AWS 認證」是專案，對應的領域是「職涯發展」。

### 3. Resources（資源）—— 感興趣的參考主題

資源是你**感興趣、可能未來用得到**的主題和參考資料。像是：

- 人工智慧最新趨勢
- 攝影構圖技巧
- 閱讀筆記和書摘
- 好用的生產力工具清單

這些東西不一定跟目前的專案直接相關，但你就是想留下來。沒關係，丟進 Resources 就對了。

### 4. Archives（檔案）—— 暫時不用的東西

已完成或目前不活躍的內容，全部移到這裡。包括：

- 已結束的專案所有資料
- 以前感興趣但現在沒在碰的主題
- 換工作後舊職位的相關文件

**Archive 不是垃圾桶，是冷儲存。** 東西放在這裡，隨時可以翻出來用，只是平時不會干擾你。

## 為什麼 PARA 跟其他方法不一樣？

很多人做筆記分類會照「主題」分：工作、生活、學習、投資……聽起來很合理，但問題是——**一則筆記往往同時屬於多個主題**，到底該放哪裡？

PARA 改照**行動性**來分：

| 層級 | 行動性 | 舉例 |
|------|--------|------|
| Projects | 最高，現在就要用 | 準備簡報的參考資料 |
| Areas | 中等，定期需要 | 每月收支追蹤表 |
| Resources | 較低，備用 | AI 工具推薦清單 |
| Archives | 最低，暫時不用 | 去年的企劃資料 |

**行動性越高，越容易找到；越不急的，自動退到背景。** 這就是 PARA 好用的關鍵。

## 五步驟開始你的 PARA 系統

### 步驟一：清空現有筆記

別急著分類。先把所有散落各處的筆記、書籤、截圖匯入同一個地方（Notion、Obsidian、Evernote 都行）。**先集中，再整理。**

### 步驟二：建立四個頂層資料夾

就四個：`1-Projects`、`2-Areas`、`3-Resources`、`4-Archives`。用數字開頭確保排序固定，簡單明瞭。

### 步驟三：逐一判斷行動性

拿起一則筆記，問自己：**「這跟目前哪個專案有關？」** 有 → 放 Projects。沒有 → **「這屬於哪個持續維護的領域？」** 有 → 放 Areas。依此類推，Resources 和 Archives。

### 步驟四：每週回顧十分鐘

每週花十分鐘快速瀏覽 Projects 資料夾：哪些專案完成了？完成就搬到 Archives。有沒有新的專案要建立？這個小習慣能讓整個系統保持活力。

### 步驟五：別追求完美分類

一則筆記不確定該放哪裡？**隨便選一個放下去就好。** PARA 的精髓是「先放再調」，永遠可以移動。完美的分類不存在，行動才是重點。

## 常見誤區

**誤區一：把 Areas 當 Projects**
「學好英文」是領域，不是專案——它沒有明確的完成標準。正確做法是把領域拆成專案：「三個月內通過多益 750 分」才是專案。

**誤區二：捨不得放 Archives**
很多人覺得東西放進 Archive 就等於消失了。其實正好相反，**移開不活躍的內容反而讓你更容易聚焦在眼前的事**。Archive 裡的東西隨時搜得到。

**誤區三：一開始就分太細**
別在每個資料夾底下又建十層子資料夾。PARA 的力量來自簡潔。**四個頂層分類就夠了**，細節在專案內部再整理。

## 開始行動吧

你不需要一口氣整理完所有筆記。今天，只要做一件事：**打開你的筆記 App，建好四個資料夾。** 然後從目前最活躍的專案開始，把相關筆記歸位。

第二大腦不是一天建成的，但只要踏出第一步，你就已經比昨天那個「存了卻找不到」的自己強太多了。

**你的知識不該被遺忘，它們值得一個好家。**
