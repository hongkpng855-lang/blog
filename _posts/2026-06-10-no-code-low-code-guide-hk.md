## 1. No-Code vs Low-Code：分別係乜？

先搞清楚定義，唔好撈亂：

### No-Code（無程式碼）

**完全唔需要寫任何 code**，全部用 drag-and-drop、視覺化編輯器、邏輯組合嚟完成。

例子：
- **Bubble**：建立完整嘅 Web App（有 database、workflow、user authentication）
- **Webflow**：建立專業網站（比 WordPress 靚 10 倍，仲好 SEO）
- **Make（舊稱 Integromat）**：串接 2000+ 款軟件嘅自動化平台
- **Airtable**：建立團隊協作 database，取代 Excel

好處：真係 0 coding，學幾日就上手
壞處：複雜邏輯有限制，Scalability 有天花板

### Low-Code（低程式碼）

**用少量 code 來完成複雜功能**，但比起傳統開發已經少咗 90% 嘅 coding 量。

例子：
- **Retool**：快速建立內部工具（公司後台、Admin Panel、Dashboard）
- **Appsmith**：開源版 Retool，可以 self-host
- **FlutterFlow**：用 drag-and-drop 建立 Mobile App，需要時寫 Dart code
- **Supabase**：開源 Firebase 替代，SQL database + Auth + Storage

好處：靈活性大好多，可以處理真正複雜嘅商業邏輯
壞處：始終要識少少 code，學習曲線比 No-Code 高

### 點樣揀？

| 你需要 | 用 |
|--------|---|
| 一個靚網站（Landing Page / 公司網站） | Webflow |
| 一個有登入功能嘅 Web App | Bubble |
| 內部管理後台（Admin Panel） | Retool / Appsmith |
| 重複工作自動化（Email / 數據同步） | Make / Zapier |
| 手機 App | FlutterFlow |
| Database + API | Supabase / Airtable |

## 2. 2026 年必知嘅 No-Code 工具生態

以下係我親自用過、推薦俾香港朋友嘅工具。我根據用途分類：

### 🌐 網站建設

**Webflow**（推薦指數：⭐⭐⭐⭐⭐）
- 最靚嘅 No-Code 網站 builder
- 內置 CMS、SEO 工具、Hosting
- 香港場景：公司官網、Landing Page、Portfolio、Blog
- 收費：Free Plan 可用，正式網站約 $15-40 USD/月
- 我話你知：用 Webflow 整嘅網站，Google PageSpeed 通常 90+ 分，WordPress 要 tune 好耐先做到

**Framer**（推薦指數：⭐⭐⭐⭐）
- 新世代網站 builder，設計自由度極高
- 內置 AI 生成網站功能
- 適合想要靚 Design 但唔識 CSS 嘅人

### 🔄 自動化工作流

**Make.com**（推薦指數：⭐⭐⭐⭐⭐）
- 前身 Integromat，比 Zapier 平至少 3 倍
- 視覺化 workflow builder，可以串接 Slack/Gmail/Notion/Google Sheets/WhatsApp 等
- 香港場景：自動回覆 email、同步 Google Sheets 數據、收到 Form submission 自動通知
- 收費：Free Plan 每月 1000 次操作，Pro $9 USD/月

### 🗄️ Database / 表單

**Airtable**（推薦指數：⭐⭐⭐⭐⭐）
- 團隊協作嘅 Database，介面靚過 Excel 100 倍
- 香港場景：客戶管理 CRM、庫存追蹤、項目管理、內容 calendar
- 內置 Forms + Automations，可以取代好多重複工作
- Free Plan 已經夠用

**Notion**（推薦指數：⭐⭐⭐⭐）
- 其實都可以當 No-Code 工具用
- 內置 Database、Forms（Typeform-like）、Automations
- 適合個人/小團隊嘅輕量應用

### 🏗️ Web App

**Bubble**（推薦指數：⭐⭐⭐⭐）
- 最強大嘅 No-Code Web App 平台
- 內置 User Authentication、Workflow Engine、Database
- 香港場景：會員系統、預約平台、Marketplace MVP、SaaS 原型
- 收費：Free Plan 可開發，正式上線約 $30-120 USD/月
- 注意：Bubble 嘅 loading speed 係最大短板，complex app 會慢

### 📱 Mobile App

**FlutterFlow**（推薦指數：⭐⭐⭐⭐）
- drag-and-drop 建立真嘅 Mobile App（唔係 wrapper）
- 可以 export 原始碼，隨時可以搵 developer 接手
- 支援 Firebase / Supabase 做 backend

## 3. 香港最實用嘅 5 大應用場景

我同幾個香港朋友夾過，以下係 No-Code 最幫到手嘅場景：

### 場景 1：Freelancer Landing Page（Webflow）

> 前後只需 2 日（Design + Content），唔使搵 developer，唔使 $500 月費

用 Webflow 建立一個靚嘅 portfolio 網站：
1. 揀 template（有好多免費嘅）
2. 改圖片、文字（drag-and-drop）
3. Set up 表單（客戶查詢自動 email 通知）
4. Publish（Webflow 自帶 hosting + SSL + CDN）

成本：Free Plan 就得，或者 $15 USD/月用 custom domain

### 場景 2：公司客户查詢自動化（Make + Airtable）

> 以前每日要用 30 分鐘手動處理查詢，而家完全自動

流程：
1. 客戶喺網站填 form → Make 觸發
2. Make 自動寫入 Airtable（建立新 record）
3. Make 自動 Send Email 確認俾客戶（用 Gmail module）
4. Make 自動通知你 Slack（附上客戶資料）
5. 你回覆咗之後，Airtable 自動更新 status

設定時間：2 小時
節省時間：每月 > 10 小時

### 場景 3：小型電商 MVP（Shopify + Airtable + Make）

> 唔使搵 developer 整網站，用一個 weekend 就上線

組合：
- Shopify 做前端（支援香港 Payment Gateway）
- Airtable 做 inventory 管理
- Make 同步訂單 → Airtable + 自動 Send Invoice

成本：Shopify Basic $25 USD/月 + Airtable Free + Make Free

### 場景 4：公司內部 IT Helpdesk（Retool）

> 取代用 Google Form + 人手 email 嘅傳統做法

用 Retool 建立：
- User 提交 ticket（form + upload screenshot）
- 自動 assign 俾相關同事
- Dashboard 睇到所有 open tickets
- Email 自動通知進度

Retool 可以駁到公司嘅 Google Workspace、Slack、甚至係舊系統嘅 API。

### 場景 5：預約管理系統（Bubble）

> 唔使俾 $2000 月費俾啲預約 SaaS

用 Bubble 建立：
- 客戶揀服務、日期、時間
- 自動確認 email
- Google Calendar integration
- Admin 後台睇 booking status

Advance 可以用 Stripe integration 收 deposit。

## 4. AI + No-Code 嘅終極組合

2026 年最大嘅改變唔係工具本身，而係 **AI 助手令 No-Code 變得更加易用**。

### 點樣用 AI 幫手？

**1. AI 生成組件**
- Bubble 內置 AI assistant，你講你想做乜，佢幫你整 UI + Workflow
- Webflow 有 AI 生成 Layout，可以 30 秒出一個設計

**2. AI 寫嘢 + 填內容**
- 用 ChatGPT / Claude 幫你寫晒網站啲 copy
- 直接講：「幫我寫一個香港 freelancer 網站嘅 Hero 標題，target 中小企客戶」

**3. AI 教點用 No-Code**
- 「Bubble 點樣整一個會員註冊系統？」- ChatGPT 會 step by step 教你
- 「Make 點樣將 Google Sheet 嘅新 row sync 去 Airtable？」- AI 俾你完整 workflow

**4. AI 幫你 Debug**
- Bubble / Make 玩到卡住咗？Screen cap 俾 AI，佢會話你知邊度出錯

### 我的實際 workflow

```
1. 用 ChatGPT 規劃 App 結構（邊啲頁面、邊啲功能）
2. 用 Webflow / Bubble 建立骨架
3. 用 AI（內置或 ChatGPT）生成內容同設計
4. 用 Make 串接晒啲 3rd party 服務
5. 用 Airtable 做 Central Database
```

用呢個組合，一個 MVP 可以由 **1-2 個月縮短到 1-2 星期**。

## 5. 由零開始嘅快速起步流程

如果你係一個完全冇 tech background 嘅香港創業者，跟呢個順序：

### Step 1：學識 Webflow（第 1-3 日）

- 去 Webflow University（免費，有片有文）
- 用一個 template 整一個簡單 portfolio 網站
- 學識：Layout、CMS、Responsive Design

### Step 2：學識 Make（第 4-5 日）

- 免費註冊 Make.com
- 整一個簡單嘅 automation：收到 email → save to Google Sheet → send Slack notification
- 學識：Modules、Filters、Router

### Step 3：學識 Airtable（第 6-7 日）

- 由一個 CRM template 開始
- 學識：Table、Linked Records、Views、Interfaces
- 將 Make 同 Airtable 連接

### Step 4：挑戰一個小 Project（第 2 星期）

- 選擇上面 5 個場景嘅其中一個
- 由頭到尾自己做一次
- 目標：唔使問人，全部自己解決

### Step 5：玩 Bubble / Retool（第 3-4 星期）

- 如果你需要更複雜嘅 Web App
- Bubble 嘅學習曲線比 Webflow 高好多，但回報都大
- 建議先完成 Step 1-3 先開始 Bubble

## 6. 常見錯誤與點避開

### ❌ 錯誤 1：一開始就用 Bubble

Bubble 係最 powerful 嘅 No-Code 平台，但亦係最複雜嘅。如果你只係想整一個簡單網站，用 Webflow 已經夠。用 Bubble 做簡單嘢等於殺雞用牛刀。

**Fix**：先確定你嘅 Use Case，再揀啱工具

### ❌ 錯誤 2：忽略 Scalability

No-Code MVP 行得通，但如果真係有人用，可能會遇到：
- Bubble loading speed 問題
- Airtable 100,000 records 上限（Free Plan 得 1,000）
- Make 嘅 operation 限額

**Fix**：一開始就用 Plan B 心態，知道咩時候要 migrate

### ❌ 錯誤 3：唔用 Version Control

No-Code 平台嘅最大弱點係難做 Version Control。一個唔小心 delete 咗嘢，未必可以還原。

**Fix**：
- 定期 export（Webflow 可以 export 成 ZIP）
- 用 Bubble 嘅 Version History
- 重要 Logic 寫低喺 Notion

### ❌ 錯誤 4：低估時間

「No-Code 即係好快」係最大嘅假象。學新工具本身就要時間，Debug 都要時間。預期第一個 Project 會比你想像中耐 2-3 倍。

**Fix**：接受學習曲線，由超簡單嘅東西開始

## 7. 進階：當 No-Code 唔夠用嘅時候

No-Code 唔係萬能。當你遇到以下情況，就要考慮 Low-Code 或者傳統開發：

1. **複雜商業邏輯**：多層 nested conditions、動態 pricing、複雜嘅 scheduling
2. **高效能要求**：大量 concurrent users、real-time data processing
3. **Custom Integration**：需要同 legacy system 或者特殊 API 整合
4. **Security / Compliance**：需要 SOC2、HIPAA 或者特定嘅 data residency

### 解決方案

- **Retool / Appsmith**：比 Bubble 更 flexible，用到 SQL
- **Supabase**：開源 Firebase，可以 host 喺香港，符合 data residency
- **Claude / ChatGPT + Coding**：AI 可以幫你寫 code，唔係完全唔掂

### Hybrid Approach（推薦）

```
No-Code（Frontend）+ Low-Code（Backend）+ AI（Glue）
```

例如：
- Webflow 做 landing page
- Supabase 做 database + authentication
- Make 做 automation
- 唔夠用嘅時候用 Claude 寫一個細細嘅 API endpoint

呢個組合嘅好處係：**每一層都用最啱嘅工具**，唔使全部嘢都迫喺一個平台。

## 8. FAQ

### Q1：真係可以完全唔寫 Code？連 HTML 都唔使識？

係，純 No-Code 可以完全唔寫 code。不過如果你識少少 HTML/CSS（或者用 AI 幫你寫），設計自由度會大好多。最基本嘅 Webflow／Bubble 可以完全冇 coding knowledge 使用。

### Q2：學邊個 No-Code 工具先最好？

由 Webflow 開始（網站），然後 Make（自動化），最後 Airtable（資料庫）。呢個組合最易上手，而且即時見到成果，好有成功感。

### Q3：No-Code 平台每個月俾錢，係咪好貴？

多數有 Free Plan。以一個生產力工具嚟講：
- Webflow $15 USD/月 ≈ $120 HKD
- Make Free Plan 夠個人用
- Airtable Free Plan 夠小型團隊
總計可能 $200 HKD/月，比搵 developer 整平 100 倍。

### Q4：No-Code 整出嚟嘅 App 可唔可以賣俾人？

理論上可以，但實際上有技術限制同 License 問題。Bubble 唔俾你用佢嘅平台整 App 賣俾人，除非你買佢嘅 Overage Plan（$115 USD/月起）。

### Q5：同傳統開發比，效能會唔會好差？

會，但要睇 Use Case。簡單嘅內部工具同 MVP 完全冇問題。如果你係做高流量嘅 consumer app，傳統開發仍然係最好嘅選擇。

### Q6：香港有冇 No-Code 嘅社群？

有㗎！Facebook 有「No Code Hong Kong」群組，仲有唔少本地嘅 Meetup。另外 Bubble 同 Webflow 嘅官方 forum 都好活躍，search "Hong Kong" 會搵到好多本地案例。

### Q7：AI 可唔可以幫我由零寫一個完整嘅 App？

可以幫你做好多，但暫時未可以完全自動化。2026 年嘅 AI（Claude、ChatGPT、GitHub Copilot）可以：
- 寫完整嘅 HTML/CSS/JS 頁面
- 生成簡單嘅 Web App
- 幫你 Debug No-Code 平台嘅 workflow

但如果你嘅 App 需要複雜嘅 backend logic、database design 同 security，AI 仍然需要人 guiding。

### Q8：用咗 No-Code 之後，第時想搵 Developer 接手會唔會好難？

視乎平台：
- **Webflow**：可以 export HTML/CSS/JS code，Developer 可以接手
- **Bubble**：專屬平台，要 Bubble developer 先接手到
- **FlutterFlow**：可以 export 完整嘅 Flutter code（真係 code）
- **Retool**：齋做 internal tool，通常唔需要轉手

如果你擔心 lock-in，FlutterFlow 同 Webflow 係最「走得甩」嘅選擇。
